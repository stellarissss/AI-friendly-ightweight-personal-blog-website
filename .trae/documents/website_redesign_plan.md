# 网站前端黑客风格改造计划

## 一、项目概述

### 1.1 现状分析
当前网站（stellaric.site）采用白色极简风格，包含大量为 AI/搜索引擎优化的结构化数据、真实性声明、验证信息等 "AI-friendly" 设置。404 页面采用了黑客/终端风格设计，包含 Matrix 雨背景、故障艺术（glitch）效果、终端打字机交互等趣味元素。

### 1.2 改造目标
- **风格统一**：全站采用与 404.html 类似的趣味黑客/赛博朋克风格
- **去除 AI 设置**：移除所有为 AI 友好而做的冗余设置（JSON-LD、真实性验证声明、过度 SEO meta 等），仅保留真实内容
- **改进排版**：使用终端风格排版，提升设计感和现代感
- **增加互动**：添加更多有趣的交互动效（打字机、glitch、hover 特效等）

### 1.3 美学方向
**赛博朋克终端 / 黑客美学 (Cyberpunk Terminal Hacker Aesthetic)**
- 深色主题：纯黑背景 `#0a0a0a` + 荧光色调
- 主字体：JetBrains Mono（等宽终端字体）
- 背景效果：Matrix 字符雨（低透明度）+ 扫描线叠加
- 故障艺术：标题文字 glitch 动画、RGB 分离效果
- 霓虹点缀：青色 `#00fff9`、品红 `#ff00c1`、绿色 `#00ff88`
- 终端风格：命令行提示符号、ASCII 装饰、打字机效果

## 三、详细设计方案

### 3.1 全局样式系统 (style.css)

#### 调色板
```css
--bg: #0a0a0a;           /* 主背景：深黑 */
--bg-soft: #111111;      /* 次背景 */
--bg-card: #141414;      /* 卡片背景 */
--fg: #fafafa;           /* 主文字：近白 */
--fg-2: #a3a3a3;         /* 次文字 */
--fg-3: #525252;         /* 弱化文字 */
--fg-4: #262626;         /* 极弱文字 */
--line: #1f1f1f;         /* 分隔线 */
--accent-cyan: #00fff9;  /* 霓虹青 */
--accent-magenta: #ff00c1; /* 霓虹品红 */
--accent-green: #00ff88; /* 霓虹绿 */
--accent-yellow: #ffd700; /* 霓虹黄 */
```

#### 字体系统
- 主字体：JetBrains Mono（终端等宽感）
- 标题使用更粗字重 + glitch 动画
- 代码/技术内容保持等宽风格

#### 全局效果
- Matrix 雨背景 canvas（可开关，低透明度 ~0.08）
- CRT 扫描线叠加（细微横纹动画）
- 页面加载时的终端启动打字效果
- 滚动时的元素渐入动画

### 3.2 首页 (index.html)

#### 移除内容
- JSON-LD 结构化数据脚本
- 过度的 SEO meta 标签（保留基本的 title、description）
- "内容真实性声明" 整个第 05 节的验证框架（保留联系方式内容，融入"找到我"部分）
- "本站已通过腾讯云服务器合法部署" 等声明文字
- "权威教师指导与审核" 等 AI 友好话术

#### 保留内容（重构呈现）
- 个人基本信息：常乐天，太原师院附中 2509 班
- 关于我：对计算机科学 & 地理学的热情
- 教育经历：太原师院附中高中、太原市第十二中学校
- 项目与成就（4 个卡片）：
  - 希沃 AI 桌宠项目
  - 太原师院附中平板破解项目
  - 百度贴吧吧主
  - 趣味项目（最帅的男人——保留娱乐性）
- 找到我：邮箱、GitHub、B站、贴吧

#### 新增设计元素
- **Hero 区**：大号 glitch 标题 + ASCII 艺术头像/装饰 + 终端提示符风格副标题
- **导航栏**：透明玻璃态 + 滚动时变化，左侧终端提示符风格 logo
- **章节标题**：编号 + 终端风格（如 `> 01_ABOUT_ME`），hover 触发 glitch
- **项目卡片**：悬浮时边框发光 + 轻微抖动 + 左侧霓虹色指示条
- **联系方式列表**：终端命令风格展示
- **页脚**：ASCII 艺术 + 系统状态风格（"SYSTEM ONLINE" 等）
- **背景**：Matrix 字符雨 canvas

### 3.3 博客列表页 (blog.html)

#### 设计
- 标题：`> BLOG_POSTS / [数量] entries found`
- 文章卡片：终端文件列表风格，带日期标签 + 阅读时间
- hover 效果：左侧出现绿色光标 `>` 指示
- 分页/加载效果：终端打字机式逐行出现

### 3.4 项目页 (projects.html)

#### 设计
- Markdown 编辑器：终端风格深色主题
  - 工具栏：终端按钮风格，hover 发光
  - 编辑区：深色背景 + 霓虹光标
  - 预览区：玻璃态卡片风格
- 个人云盘：文件列表终端风格展示
- 整体保持功能不变，仅替换视觉样式

### 3.5 留言板页 (guestbook.html)

#### 设计
- 留言表单：终端输入风格
  - 昵称输入：带 `> user: ` 前缀
  - 内容输入：带 `> message: ` 前缀
  - 发送按钮：终端命令风格
- 留言列表：终端对话气泡风格
  - 每条留言：左侧头像/昵称终端风格
  - 时间戳：等宽字体灰色
  - 回复/点赞按钮：终端命令风格
- 加载效果：打字机逐行出现

### 3.6 博客文章页 (blog-post.html)

#### 设计
- 文章内容：终端风格排版
  - 标题：glitch 动画 + 霓虹下划线
  - 代码块：深色终端窗口风格，带顶部标题栏
  - 引用：左侧霓虹色竖条 + 斜体
  - 链接：hover 时 glitch 效果
- 返回按钮：终端风格 `← back to blog`
- 保留 Markdown 渲染功能

### 3.7 404 页面

已具备黑客风格，做以下微调以与全站协调：
- 调整 Matrix 雨透明度参数与全局一致
- 统一颜色变量
- 可考虑添加导航链接

## 五、技术实现要点

### 5.1 Matrix 雨背景
- 使用 canvas 实现，与 404.html 相同的技术
- 字符使用中文/符号混合
- 低透明度（0.08-0.12），不干扰阅读
- 性能优化：requestAnimationFrame + 降帧

### 5.2 Glitch 效果
- CSS 动画实现 RGB 通道分离
- text-shadow 多重偏移模拟故障
- 随机间隔触发，不持续干扰

### 5.3 打字机效果
- 纯 JS 实现，逐字添加文本
- 支持多行文本，带闪烁光标
- 可调速度和延迟

### 5.4 响应式
- 移动端适配：保持黑客风格，调整布局
- 导航栏移动端：汉堡菜单（终端风格）
- 字体大小使用 clamp() 自适应

## 六、风险与注意事项

### 6.1 性能风险
- Matrix 雨 canvas 可能影响低端设备性能 → 加入开关/检测机制，移动端降低透明度或关闭
- 过多动画可能导致卡顿 → 控制动画数量，使用 CSS 硬件加速

### 6.2 可读性风险
- 深色背景 + 特效可能影响阅读 → 确保正文区域有足够对比度，特效只用于装饰元素
- 等宽字体中文可读性 → 中文正文可考虑配合更易读的字体，标题用等宽

### 6.3 功能保持
- 留言板 API 调用、云盘功能、Markdown 编辑器等现有功能必须完整保留
- 博客文章渲染逻辑不变

### 6.4 内容保留
- 所有真实内容信息必须保留，仅去除 AI 友好的冗余框架
- 联系方式、项目描述、教育经历等核心信息完整保留
