# 网站前端界面现代化改造计划

## 一、现状分析

### 当前网站概况
- **站点类型**：个人博客/主页（学生开发者常乐天）
- **技术栈**：纯 HTML/CSS/JS，无框架
- **页面数量**：6 个页面（首页、博客列表、博客详情、项目、留言板、404）
- **当前设计风格**：黑白极简主义，使用 Inter + JetBrains Mono + Noto Serif SC
- **现有交互**：基础 hover 效果、淡入动画、留言板功能
- **亮点**：404 页面已具备独特的黑客终端风格，是很好的设计参考

### 可提升空间
1. **字体选择**：Inter 过于通用，缺乏个性
2. **色彩层次**：纯黑白略显单调，缺少点睛的强调色
3. **排版节奏**：段落间距、字号层级可以更有呼吸感
4. **交互丰富度**：hover 效果单一，缺少滚动触发动画、微交互
5. **视觉细节**：背景纹理、装饰元素、卡片设计可以更精致
6. **布局变化**：可以打破单一纵向流，增加一些不对称设计

---

## 二、设计方向

### 设计理念：「编辑风极简 + 暖色人文感」

**核心概念**：在保持极简主义的基础上，注入杂志般的编辑排版质感和温暖的人文气息。不是冰冷的科技极简，而是有温度、有个性的精致极简。

### 为什么是这个方向？
- 站长是学生开发者 + 地理爱好者，兼具理性与人文气息
- 纯黑白显得过于冷峻，加入暖色点缀更有亲和力
- 杂志编辑风提升内容的高级感和可读性
- 与 404 页的极客风格形成互补但不冲突

## 三、具体改造方案

### 3.1 字体系统（Typography）

**目标**：摒弃通用的 Inter，选择更有性格的字体组合

| 用途 | 字体 | 理由 |
|------|------|------|
| 标题/显示字体 | **Fraunces** (可变字体) | 精致的衬线字体，有人文气息，可变字重提供丰富层次 |
| 正文无衬线 | **Sora** 或 **Manrope** | 现代几何无衬线，比 Inter 更有个性，同时保持可读性 |
| 等宽字体 | **JetBrains Mono**（保留） | 现有字体已很好，继续沿用 |
| 中文衬线 | **Noto Serif SC**（保留） | 与 Fraunces 的人文气质匹配 |

**字号层级优化**：
- H1: 更大更有气势（clamp(2.2rem, 6vw, 3.5rem)）
- H2: 增加字号和字重对比
- 正文: 略微增大行高，提升阅读舒适度
- 引入字号比例关系（1.25 或 1.333 模块化比例）

### 3.2 色彩系统（Color Palette）

**目标**：从纯黑白升级为「暖白 + 墨黑 + 赭石强调色」

```css
--bg:         #faf8f5;    /* 暖白色背景，代替冷白 */
--bg-soft:    #f3efe9;    /* 暖米色软背景 */
--bg-card:    #ffffff;    /* 卡片纯白 */
--fg:         #1a1613;    /* 墨黑色，代替纯黑 */
--fg-2:       #4a423d;    /* 深棕灰 */
--fg-3:       #7a726d;    /* 中棕灰 */
--fg-4:       #a8a09b;    /* 浅棕灰 */
--line:       #e8e2db;    /* 暖灰色边线 */
--line-2:     #d6cfc7;    /* 深一点的边线 */

/* 强调色：赭石/陶土色 —— 温暖、有质感、不刺眼 */
--accent:     #b85c38;    /* 主强调色：赭石色 */
--accent-soft:#e8d5c8;    /* 浅赭石，用于背景点缀 */
--accent-2:   #2d4a3e;    /* 辅助色：深松绿，偶尔用于对比 */
```

**色彩使用原则**：
- 90% 黑白灰（暖调）
- 8% 强调色用于关键交互（链接 hover、按钮、装饰元素）
- 2% 辅助色用于惊喜时刻

### 3.3 排版与布局（Layout & Composition）

#### 布局优化
- **最大宽度**：从 760px 扩展到 860px，提升阅读舒适度
- **留白系统**：建立 8px 为基础单位的间距系统
- **Hero 区域**：更有仪式感的入场，大号标题 + 装饰元素
- **章节标题**：左侧竖线装饰 + 编号，增强编辑感
- **卡片设计**：
  - 增加细微的阴影和圆角（2px 极小圆角）
  - hover 时微妙上浮 + 阴影加深
  - 左侧赭石色竖线标识

#### 首页特别优化
- Hero 区域增加大号装饰数字或年份背景（低透明度）
- 项目卡片改为网格布局（桌面端两列）
- 「找到我」部分改为图标 + 文字的水平排列

### 3.4 动画与交互（Motion & Interaction）

#### 入场动画
- **页面加载**：分阶段交错淡入（staggered reveal）
- **滚动触发**：元素进入视口时从下方滑入淡入
  - 使用 Intersection Observer 实现
  - 不同类型元素有不同的延迟和动效曲线

#### Hover 微交互
- **链接**：下划线从左向右滑动 + 颜色渐变到赭石色
- **卡片**：
  - 向上浮动 4px
  - 阴影柔和加深
  - 左侧赭石色线条延伸动画
  - 标题颜色微变
- **按钮**：
  - 背景色填充动画
  - 微妙的 scale(0.98) 按压效果
- **导航链接**：下划线动画 + 颜色过渡

#### 特色交互
1. **自定义光标**（可选，桌面端）
   - 小圆点跟随鼠标
   - hover 到可点击元素时放大变色

2. **滚动进度条**
   - 页面顶部细细的赭石色进度条
   - 随滚动进度填充

3. **文字选中效果**
   - 赭石色背景 + 白色文字
   - 代替默认的蓝色选中

4. **图片渐入**（如有图片）
   - 模糊占位 → 清晰加载的效果

5. **列表项序号动画**
   - 滚动到时数字从零计数到目标值

### 3.5 视觉细节（Visual Details）

#### 背景与纹理
- 页面背景：极淡的噪点纹理（noise texture），增加质感
- 或：极淡的渐变（暖白到更暖的米白）
- 卡片背面：细微的内阴影，营造纸张感

#### 装饰元素
- 章节标题旁的几何装饰线
- 页面角落的细微装饰（页码式设计）
- 引用块：左侧粗赭石色线 + 引号装饰

#### 导航栏
- 增加 scroll 时的背景变化（更不透明 + 阴影）
- Brand 左边的小圆点改为赭石色，呼吸动画

#### 页脚
- 增加设计感的分割线
- 更精致的排版布局

## 四、各页面改造重点

### 4.1 首页 (index.html)
- **Hero 区域**：更大的标题，装饰性背景元素，更有冲击力的入场
- **关于我**：首字母下沉效果（drop cap），增加杂志感
- **教育经历**：时间线样式优化
- **项目与成就**：卡片网格布局（桌面 2 列），更丰富的 hover 效果
- **找到我**：图标化展示，水平排列，hover 有跳动效果
- **真实性声明**：引用框样式优化

### 4.2 博客列表页 (blog.html)
- 文章卡片增加摘要预览行数限制
- 增加文章标签/分类展示
- 阅读时间估算显示
- 卡片 hover 时箭头动画

### 4.3 项目页 (projects.html)
- Markdown 编辑器样式全面升级
- 工具栏更精致的按钮设计
- 编辑器边框和阴影优化
- 云盘文件列表卡片化

### 4.4 留言板 (guestbook.html)
- 表单设计升级，更有质感的输入框
- 留言卡片增加头像占位（字母头像）
- 点赞按钮动画效果
- 回复按钮展开动画

### 4.5 博客详情页 (blog-post.html)
- 文章标题更大更有气势
- 增加作者信息和日期的元信息区域
- 目录导航（如标题较多时，可选）
- 阅读进度条
- 首尾装饰元素

### 4.6 404 页面
- 保留现有终端风格（已经很有特色）
- 微调配色使其与整体暖色调更协调
- 增加更多有趣的命令彩蛋

## 五、技术实现方案

### 文件修改清单

| 文件 | 改动类型 | 说明 |
|------|----------|------|
| `style.css` | **重写** | 全面的样式系统重构，是改动最大的文件 |
| `index.html` | 修改 | 调整部分结构，增加动画触发元素 |
| `blog.html` | 修改 | 文章卡片结构微调 |
| `projects.html` | 修改 | 编辑器和云盘样式优化 |
| `guestbook.html` | 修改 | 留言卡片和表单样式优化 |
| `blog-post.html` | 修改 | 文章页布局调整 |
| `404.html` | 微调 | 配色微调 |

### 新增功能（JS）
1. 滚动触发动画（Intersection Observer）
2. 滚动进度条
3. 自定义光标（可选）
4. 导航栏滚动效果

### CSS 架构
- 使用 CSS 变量统一管理所有设计 token
- 保持 BEM 风格的类命名
- 移动端响应式优化
- 尊重 `prefers-reduced-motion` 设置

## 六、风险与注意事项

### 潜在风险
1. **字体加载**：新字体可能影响首屏性能 → 使用 `font-display: swap` + 预连接
2. **过度动画**：动画太多可能影响体验 → 克制使用，尊重减少动画偏好
3. **兼容性**：新 CSS 特性 → 提供优雅降级
4. **加载速度**：背景纹理等资源 → 使用 CSS 生成或极小的 base64 图片

### 保留原则
- 所有现有功能（留言板、Markdown 编辑器、云盘等）保持可用
- SEO 相关的 meta 标签和结构化数据保持不变
- 不引入额外的大型依赖库
- 移动端体验不降级
