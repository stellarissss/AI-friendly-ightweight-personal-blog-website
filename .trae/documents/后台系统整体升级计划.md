# 后台系统整体升级计划

## 摘要

将现有的单页 `admin.php`（仅支持发布博客）升级为一个带顶部 Tab 切换的现代后台，包含三大模块：
**撰写博客 / 博客管理 / 文件管理**。博客模块新增「列表查看、修改正文、删除文章、在线预览」能力，并在发布或修改后自动推送百度；文件模块对接 `projects.html` 的个人云盘，支持上传、列表、下载、删除。同时重写 `push.php` 作为百度推送的统一入口与手动工具。

---

## 现状分析

### 现有后台 [admin.php](file:///workspace/admin.php)
- 单一 PHP 文件，仅含「撰写新博客」表单（标题 / slug / 简介 / 正文 + Markdown 实时预览）。
- 发布逻辑：保存 `posts/<slug>.md` → 用正则在 [blog.html](file:///workspace/blog.html) 的 `<!-- BLOG_LIST_START -->...<!-- BLOG_LIST_END -->` 锚点间插入新卡片 → 内联 curl 推送百度。
- 登录靠 session + 明文密码比对，退出走 `?logout=1`。
- **缺失**：无法查看已有文章、无法修改、无法删除、无在线预览；UI 无导航分区。

### 云盘相关
- [projects.html](file:///workspace/projects.html) 的「个人云盘」区块调用 `/list.php`（返回 `[{name,size,time}]`）和 `/download.php?f=<name>`。
- 这两个 PHP 文件**在本地仓库中不存在**（仅存在于生产服务器 119.45.14.187），需要在本仓库新建以保证服务端与线下同步。
- **缺失**：无上传入口、无删除入口。

### push.php
- README 与 `重要!blog工作流.txt` 都提到 `https://stellaric.site/push.php`，但本地仓库不存在。需要新建并重写其逻辑，作为发布/修改后推送百度的统一能力。

### 样式与约定
- 全站使用 [style.css](file:///workspace/style.css) 的黑白极简风（CSS 变量 `--fg/--bg/--line/--font-mono` 等），后台应复用以保持一致。
- 博客卡片结构固定：
  ```html
  <article class="card">
      <div class="meta">YYYY / MM / DD · POST</div>
      <h3><a href="blog-post.html?file=posts/<slug>.md">标题</a></h3>
      <p>简介</p>
  </article>
  ```

## 拟定改动

### 1. 重构 [admin.php](file:///workspace/admin.php)（核心改动）

**结构**：保留登录/退出逻辑不变。登录后页面改为顶部 Tab 布局，通过 `?tab=write|posts|files` 或前端 JS 切换三个面板。所有写操作仍走 POST，受现有 session 保护。

**Tab A — 撰写博客**（保留并优化现有功能）
- 复用现有表单（标题 / slug / 简介 / 正文 + marked.js 实时预览）。
- 发布成功后调用 `push.php` 的推送函数（见下）自动推送百度，并在消息里展示返回结果。
- 新增「保存为草稿」可选（仅写 md，不更新 blog.html、不推送）——若实现成本低则加，否则跳过。

**Tab B — 博客管理**（新增）
- 扫描 `posts/*.md`，列出表格：文件名（slug）、从正文第一个 `# 一级标题` 提取的标题、文件修改时间、操作按钮（编辑 / 预览 / 删除）。
- **编辑**：点击后进入「撰写博客」表单的编辑模式，回填 slug（只读，作为唯一键）、标题（取自 md 的 H1）、简介（从 blog.html 对应卡片的 `<p>` 解析）、正文（读取 md 文件）。保存时：
  - 覆盖写 `posts/<slug>.md`；
  - 用正则按 `href="blog-post.html?file=posts/<slug>.md"` 定位 blog.html 中的对应卡片，更新其标题与简介文本（保留日期）；
  - 调用 `push.php` 推送该文章 URL 到百度；
  - 提示成功。
- **预览**：点击「预览」用 marked.js 在页面内弹层（modal）渲染当前 md 正文，复用 `#markdown-content` 的样式。
- **删除**：二次确认后删除 `posts/<slug>.md`，并从 blog.html 中移除对应卡片块（正则匹配整段 `<article class="card">...</article>` 中含该文件名的那一块）。删除不推送百度。

**Tab C — 文件管理**（新增，对接云盘）
- 上传表单（`enctype="multipart/form-data"`，POST `action=upload`）：保存到 `uploads/` 目录（目录不存在则 `@mkdir(..., 0775, true)`）。上传后刷新列表。
- 文件列表：读取 `uploads/` 目录，展示文件名、大小、修改时间，每行带「下载 / 删除」按钮。
  - 下载：直接链接 `download.php?f=<name>`。
  - 删除：POST `action=delete_file&name=<name>`，二次确认后 `unlink`。
- 该列表与 `projects.html` 公开云盘共用同一个 `uploads/` 目录与 `list.php`，保证后台删除后前端同步消失。

**安全**：
- 所有写操作（publish / edit / delete_post / upload / delete_file）均在 session 登录校验之后，沿用现有保护。
- 文件名一律 `basename()` 过滤，拒绝含 `/`、`\`、`..` 的名称，防路径穿越。
- 上传仅允许常规文件类型（按扩展名白名单或仅限非可执行类），文件名冲突时重命名。

**UI**：顶部 Tab 用按钮组，激活态下加下划线（复用 `.nav-links a::after` 思路）。内联少量 CSS，复用 `var(--fg/--bg/--line/--font-mono)`。移动端 Tab 横向可滚动。

### 2. 新建 [push.php](file:///workspace/push.php)（重写百度推送）

- 顶部同样接 session 登录校验（与 admin.php 一致），未登录跳转 admin.php。
- 定义函数 `push_to_baidu($urls)`：封装 curl 调用 `http://data.zz.baidu.com/urls?site=...&token=...`，参数与现有 admin.php 内联逻辑一致，返回百度响应文本。
- `admin.php` 通过 `require 'push.php'` 复用此函数（发布/修改后调用），避免重复代码。
- 直接访问 `push.php` 时展示一个简易 UI：列出 `posts/*.md`，每篇带「推送」按钮（GET `?file=<slug>` 或 POST），并可「一键推送全部」。推送结果回显。
- 站点与 token 写成顶部常量，便于后续修改。

### 3. 新建 [list.php](file:///workspace/list.php)（公开，云盘列表接口）

- 无需登录（projects.html 公开调用）。
- 扫描 `uploads/` 目录，输出 JSON：`[{"name","size","time"}, ...]`，字段与 projects.html 现有 `formatSize/formatDate` 消费方式一致（time 为 Unix 时间戳秒）。
- 跳过 `.`、`..` 及子目录。

### 4. 新建 [download.php](file:///workspace/download.php)（公开，云盘下载接口）

- 无需登录。
- 取 `$_GET['f']`，`basename()` 过滤并校验不含路径分隔符，拼接到 `uploads/` 下；文件不存在返回 404。
- 设置 `Content-Type: application/octet-stream`、`Content-Disposition: attachment; filename="..."`，`readfile` 输出。

### 5. 不改动的文件
- [projects.html](file:///workspace/projects.html)：现有 `/list.php`、`/download.php?f=` 调用与新接口完全兼容，无需改动；公开页保持只读。
- [blog.html](file:///workspace/blog.html) / [blog-post.html](file:///workspace/blog-post.html) / [style.css](file:///workspace/style.css)：不改。
- `posts/` 现有文章不动。

## 假设与决策

1. **云盘存储目录**：统一使用 `uploads/`（相对网站根目录）。生产服务器上若已有其他目录，部署时调整常量即可。
2. **list.php / download.php 公开**：projects.html 是公开页，必须可匿名访问；上传与删除仅在 admin.php 内完成，受 session 保护。
3. **博客元数据**：不引入额外的 meta.json，标题从 md 的首个 H1 提取，简介从 blog.html 卡片的 `<p>` 解析回填，保持现有「md 文件 + blog.html 卡片」双写模型。
4. **推送站点/Token**：沿用 admin.php 现有的 `site=stellaric.site` 与 `token=4dbTytHYWwc4Zhx1`，提取为 push.php 顶部常量。
5. **密码**：保持现有明文 session 方案不变（不在本次升级范围内引入哈希/数据库）。
6. **编辑模式下的 slug**：设为只读，作为定位键；如需重命名 slug，等价于「新建+删除」，本次不单独支持。
7. **删除文章/文件**：均做前端二次确认（`confirm()`），后端再做一次存在性校验。

## 验证步骤

1. **登录**：访问 `admin.php`，未登录显示登录页；输错密码提示错误；输对进入后台。
2. **Tab 切换**：三个 Tab 可正常切换，URL/状态保持。
3. **发布博客**：填写后发布 → `posts/` 生成 md → `blog.html` 锚点间出现新卡片 → 提示百度推送返回 → 打开 `blog.html` 与 `blog-post.html?file=posts/<slug>.md` 验证。
4. **博客管理列表**：列出 `posts/` 全部 md，标题/时间正确。
5. **编辑博客**：回填字段正确 → 改正文与简介 → 保存 → md 被覆盖 → blog.html 对应卡片标题/简介更新 → 百度推送返回。
6. **在线预览**：弹层正确渲染 Markdown。
7. **删除博客**：确认后 md 被删 → blog.html 对应卡片消失 → 列表刷新。
8. **文件上传**：在文件管理 Tab 上传一个文件 → `uploads/` 出现该文件 → 列表刷新。
9. **文件下载**：点下载 → 浏览器收到原文件。
10. **文件删除**：确认后 `uploads/` 中文件被删 → 列表刷新 → 同时访问 `projects.html` 云盘区块确认该文件消失（因共用 list.php/uploads/）。
11. **list.php**：直接 `curl /list.php` 返回合法 JSON。
12. **download.php 路径穿越**：`/download.php?f=../admin.php` 等应被拒绝（404 或报错），不可越权读取。
13. **push.php**：直接访问展示推送 UI；点「推送」对指定文章推送百度并回显结果；未登录跳转 admin.php。
14. **移动端**：Tab 与表单在窄屏下可用。
